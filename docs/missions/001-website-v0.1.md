# Mission 001 — Website v0.1

## Contexte

VestigiumLabs dispose déjà d'une identité initiale :

- un nom ;
- un logo ;
- une bannière ;
- un README ;
- une devise ;
- une vision générale.

Le site web doit maintenant devenir la première démonstration concrète de la méthode VestigiumLabs.

Cette mission sert à concevoir et mettre en œuvre une première version simple du site officiel.

---

## Objectif

Créer une première page d'accueil statique pour VestigiumLabs.

Le visiteur doit comprendre rapidement :

- ce qu'est VestigiumLabs ;
- les domaines couverts ;
- l'esprit du laboratoire ;
- où trouver les futurs projets et publications.

---

## Périmètre

Cette mission inclut :

- une page `index.html` ;
- une feuille de style CSS ;
- un éventuel fichier JavaScript minimal ;
- l'intégration du logo ;
- l'intégration de la bannière ou de son identité visuelle ;
- une structure responsive simple ;
- une documentation minimale des choix.

---

## Hors périmètre

Cette mission exclut :

- tout framework ;
- tout système de blog ;
- toute base de données ;
- toute authentification ;
- toute animation complexe ;
- tout backend ;
- toute optimisation avancée ;
- toute publication automatique.

---

## Contraintes

- Utiliser HTML5, CSS3 et JavaScript simple si nécessaire.
- Garder une structure compréhensible.
- Respecter l'identité visuelle existante.
- Privilégier la simplicité.
- Ne pas complexifier pour un résultat équivalent.
- Le Directeur de laboratoire ne doit pas écrire de code.
- Chaque livrable important doit être validé avant l'étape suivante.

---

## Livrables attendus

- Spécification de l'Architect.
- Proposition visuelle du Designer.
- Contenu rédactionnel du Writer.
- Implémentation du Developer.
- Rapport de vérification du Reviewer.
- Rapport de publication du Publisher.

---

## Critères de réussite

La mission est réussie lorsque :

- la page d'accueil existe ;
- le logo est intégré ;
- l'identité VestigiumLabs est respectée ;
- le site est lisible sur ordinateur et mobile ;
- le contenu est clair ;
- le code reste simple ;
- le dépôt est propre ;
- le résultat peut être publié sur Cloudflare Pages ou GitHub Pages.
