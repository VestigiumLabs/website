# Mission 002 — Design System v1

## Contexte

L'architecture de la première version du site web de VestigiumLabs a été validée.

Le laboratoire dispose déjà d'une identité graphique composée :

- d'un logo officiel ;
- d'une bannière officielle.

Le Designer est chargé de transformer cette identité en un système graphique cohérent, réutilisable et documenté.

Ce Design System constituera la référence visuelle du laboratoire pour le site web ainsi que pour les futurs projets.

---

## Objectif

Concevoir le premier **Design System** de VestigiumLabs.

Le résultat doit permettre de définir les règles graphiques officielles du laboratoire afin d'assurer une identité visuelle cohérente, durable et facilement réutilisable.

---

## Périmètre

Cette mission inclut :

- l'analyse du logo existant ;
- l'analyse de la bannière existante ;
- la définition de la palette de couleurs officielle ;
- le choix des typographies ;
- la définition de la hiérarchie typographique ;
- la définition des espacements ;
- la définition des composants graphiques principaux ;
- la définition des règles d'utilisation du logo ;
- la définition des règles d'utilisation de la bannière ;
- la conception d'une maquette desktop de la page d'accueil ;
- la conception d'une maquette mobile de la page d'accueil ;
- la documentation des choix graphiques.

---

## Hors périmètre

Cette mission exclut :

- le développement HTML ;
- le développement CSS ;
- le développement JavaScript ;
- l'intégration technique ;
- la rédaction des contenus du site ;
- le référencement (SEO) ;
- les animations complexes ;
- toute modification du logo officiel ;
- toute modification de la bannière officielle.

---

## Contraintes

- Respecter l'identité graphique existante.
- Ne pas modifier le logo officiel.
- Ne pas modifier la bannière officielle.
- Privilégier un design sobre, professionnel et intemporel.
- Concevoir une interface adaptée aux ordinateurs et aux appareils mobiles.
- Favoriser la lisibilité et l'accessibilité.
- Éviter les effets graphiques sans bénéfice fonctionnel.
- Documenter les choix graphiques importants.
- Produire un résultat facilement exploitable par le Developer.

---

## Livrables attendus

Le Designer devra produire :

- un document **Design System v1** ;
- une palette de couleurs officielle ;
- un guide typographique ;
- une bibliothèque de composants graphiques ;
- une maquette desktop de la page d'accueil ;
- une maquette mobile de la page d'accueil ;
- un guide d'utilisation de l'identité graphique.

---

## Critères de réussite

La mission est réussie lorsque :

- une identité graphique cohérente est définie ;
- les règles graphiques sont documentées ;
- les composants graphiques sont homogènes ;
- les maquettes respectent l'architecture validée ;
- le Design System peut servir de référence aux futurs projets du laboratoire.
